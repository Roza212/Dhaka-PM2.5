# LLM Prompt Library

The following iterative system prompts were utilized to generate the end-to-end architecture:
1. **Core Documentation**: "Build this component completely: Generate the core project documentation including PRD.md, TRD.md, SRS.md..."
2. **Directory Structure**: "Build this component completely: Create the standardized directory structure for our project..."
3. **Security Initialization**: "Build this component completely... Target Output: SECURITY.md, THREAT_MODEL.md, and .env.example."
4. **API Spec**: "Target Output: Exhaustive functional requirement documentation and initial API schema definitions."
5. **ETL Pipeline**: "Target Output: Complete ETL_SPEC.md and automated ETL pipeline scripts inside the data/ directory."
6. **Graph Builder**: "Target Output: Graph construction scripts that map out nodes and edges for network processing."
7. **Feature Engineering**: "Target Output: FEATURE_SPEC.md and the final feature engineering code using optimal selection methods."
8. **ML STGCN Pipeline**: "Target Output: 1) Baseline model implementations and the primary STGCN architecture; 2) Full MLflow integration..."
9. **FastAPI Backend**: "Target Output: A complete backend web application using FastAPI to serve model predictions... strict API security measures..."
10. **Frontend GIS**: "Target Output: A complete GIS dashboard with interactive map rendering and data charts."
11. **DevOps/Infrastructure**: "Target Output: Dockerfiles, Kubernetes manifests, complete CI/CD GitHub Actions pipelines, and observability setup..."
12. **QA & Documentation**: "Target Output: USER_GUIDE.md, API_GUIDE.md, MODEL_CARD.md, DATA_CARD.md, DEPLOYMENT_GUIDE.md, RUNBOOK.md, and PROMPT_LIBRARY.md."
